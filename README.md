# 📊 Análise de Vendas com SQL

## 🧠 Objetivo

Este projeto tem como objetivo explorar e analisar dados de vendas de um ambiente de varejo, buscando identificar padrões de comportamento, desempenho de produtos, lucratividade e impacto de estratégias de desconto.

A análise foi conduzida com foco em gerar **insights de negócio**, indo além da simples manipulação de dados.

---

## 🛠️ Ferramentas utilizadas

* SQL (MySQL)
* Modelagem de dados relacional
* Análise exploratória de dados (EDA)

---


## 🧩 Base de dados

O conjunto de dados simula um cenário de varejo com:

* Clientes (dados demográficos)
* Produtos (categoria, preço e custo)
* Lojas (localização)
* Transações (vendas, quantidade, desconto)

Estrutura em modelo relacional (semelhante a esquema estrela), permitindo análises mais completas com JOINs.

---

## 🔍 Análises realizadas

* Total de vendas
* Produto mais vendido
* Receita por produto
* Lucro por produto
* Vendas por loja
* Impacto dos descontos na receita
* Impacto dos descontos no lucro
* Lucro médio por transação
* Ranking de clientes por receita

---

## 💡 Principais insights

### 📌 Volume ≠ Lucratividade

Produtos com maior volume de vendas não são necessariamente os mais lucrativos.

> Isso indica a necessidade de estratégias focadas em margem, e não apenas em volume.

---

### 📌 Impacto dos descontos

* Descontos mais altos aumentam o volume de vendas e o lucro total
* Descontos moderados apresentam melhor eficiência por transação
* O impacto do desconto não é linear

> Ou seja: não basta dar desconto — é preciso definir a intensidade correta.

---

### 📌 Estratégia de negócio

* Para maximizar lucro total → descontos maiores podem ser utilizados
* Para otimizar margem → descontos moderados são mais eficientes

---

### 📌 Clientes mais valiosos

Foi possível identificar os clientes que mais geram receita, permitindo futuras estratégias como:

* Programas de fidelidade
* Segmentação de clientes
* Ações direcionadas de marketing
